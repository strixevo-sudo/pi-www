From FiveM Dev to Self-Hosted Website in a Day

23 Mar 20264 min readby STRIXEVO

People see a website like this and assume the person who built it has some kind of formal web development background. Computer science degree, bootcamp, years of grinding tutorials. I have none of that.

What I do have is years of FiveM development - building scripts, UI panels and server resources for GTA roleplay servers. If you've done FiveM dev you already know Lua, JavaScript and CSS. You just never pointed any of it at a real website before.

That's exactly where I was. I knew the code. I just had no idea how to actually get a website on the internet.

What FiveM actually teaches you

FiveM development is more serious than people give it credit for. You're writing Lua for server and client side logic, JavaScript for NUI interfaces, and CSS for styling those interfaces. You deal with events, callbacks, state management, API calls - proper programming concepts.

The UI work especially translates directly. A FiveM NUI panel is just an HTML page running inside the game engine. If you've built one of those, you've already built a webpage - you just never put it on the internet.

// The gap Knowing how to write HTML and CSS is only half the job. The part nobody tells you about is how to actually serve those files to the rest of the world. That's where most people get stuck - including me.

The bit I didn't know

I could write the HTML. I could style it. I could add JavaScript. What I couldn't do was answer the question - how does someone in another country actually see this file on my PC

The answer turned out to be three things: Nginx to serve the files, Cloudflare Tunnel to bypass my ISP's CGNAT and get traffic to my machine without port forwarding, and a domain name so people have something to type in the address bar.

The full breakdown of how I set all that up is in my other post - How I Set Up My Self-Hosted Website From Scratch. That covers every command from start to finish.

Building the site itself

Once the server side was sorted, the actual site came together fast. The FiveM background meant I wasn't learning HTML and CSS from scratch - I was just applying what I already knew in a slightly different context. No game engine wrapper, no NUI restrictions, just a browser and a blank page.

I used Claude to help put it all together - bouncing ideas, generating components, working through the JavaScript. Think of it less like asking an AI to do your homework and more like having a senior dev sat next to you who you can ask anything. The ideas, the direction and the content are all mine. Claude helped me move faster.

What I'd tell anyone in the same position

If you've done any kind of scripting or modding - FiveM, Minecraft plugins, Roblox, whatever - you're closer to being able to build a website than you think. The code isn't the hard part. It's the server stuff that feels alien at first.

But it's learnable. I went from no website to a live self-hosted site on my own domain in a single day. The whole thing cost GBP 1.97. There's genuinely no excuse not to have your own corner of the internet.
