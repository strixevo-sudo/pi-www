How I Migrated My Website to a Raspberry Pi 3

25 Mar 20267 min readby STRIXEVO

A few days ago I posted that the site was moving to a Raspberry Pi 3 my brother gave me. That migration is now done. Here's exactly how I did it - every command, every problem I hit, and how it all ended up working.

// What this covers Setting up Raspberry Pi OS, installing Nginx, migrating all the site files from the old VM, wiring up the Cloudflare Tunnel, and hardening the server. The full process from a blank SD card to a live site.

Why move off the gaming PC

The site was running inside a Kali Linux VM on my main gaming PC. It worked - but it meant every time I shut the PC down, the site went offline. Not exactly reliable hosting. The Raspberry Pi 3 pulls around 3 watts under load, runs Linux natively with no VM overhead, and can sit in the corner of my room running 24/7 without making a sound or adding anything meaningful to the electric bill.

The hardware was already there. My brother handed it to me months ago and it had been sat in a drawer ever since. Time to actually use it.

Step 1 - Flash Raspberry Pi OS

Downloaded Raspberry Pi Imager on Windows, selected Raspberry Pi OS Lite (64-bit) - no desktop environment needed, this is a server - and flashed it to a microSD card.

Before ejecting, I used the Imager's advanced settings to pre-configure a few things: set a hostname, enabled SSH, and set a username and password. This means the Pi comes up SSH-ready on first boot with no monitor or keyboard needed - headless from the start.

# Once booted, SSH in from Windows
ssh [USERNAME]@[PI_IP]
// Headless setup tip Always use Raspberry Pi Imager's OS customisation options before flashing. Enabling SSH and setting credentials there saves you connecting a monitor just to get the Pi on the network.

Step 2 - First boot setup

Once I was in via SSH, standard first steps - update everything and set the timezone:

sudo apt update && sudo apt upgrade -y
sudo timedatectl set-timezone Europe/London

Step 3 - Install Nginx

Same as on the Kali VM - Nginx is what actually serves the HTML files to anyone who visits the site:

sudo apt install nginx -y
sudo systemctl enable nginx
sudo systemctl start nginx

Hitting the Pi's local IP in a browser at this point shows the default Nginx welcome page. That means it's working.

Step 4 - Copy the site files across

All the site files were on my Windows machine - I'd backed them up from the old VM before decommissioning it. Transferring them to the Pi was one SCP command from PowerShell:

scp -r "$env:USERPROFILE\Desktop\website-backup\html\*" [USERNAME]@[PI_IP]:/var/www/html/

That copies every file from the backup folder straight into the Nginx web root on the Pi. Clean URLs, blog posts, CSS, JavaScript, everything - all transferred in one go.

// ProtonVPN note If you're running a VPN on Windows and SCP keeps throwing a Permission denied error, that's your VPN routing the traffic away from the local network. Turn it off before running the transfer, or add a static route so [LOCAL_SUBNET] bypasses the tunnel.

Step 5 - Fix file permissions

After the transfer, Nginx needs to be able to read everything in the web root:

sudo chown -R www-data:www-data /var/www/html
sudo chmod -R 755 /var/www/html

Step 6 - Configure Nginx

I needed clean URLs - no .html extensions showing in the address bar - and the same hardening from the old server. Edited the default site config:

sudo nano /etc/nginx/sites-available/default

Key additions to the location block:

# Clean URLs
try_files $uri $uri.html $uri/ =404;

# Hide Nginx version
server_tokens off;

# Block hidden files
location ~ /\. { deny all; }
sudo nginx -t
sudo systemctl reload nginx

Step 7 - Install Cloudflare Tunnel

This is what gets the site on the internet without needing to open any ports on the router. My ISP uses CGNAT so port forwarding isn't an option - the tunnel is the solution. The Pi connects outbound to Cloudflare's network, Cloudflare handles all incoming traffic.

Raspberry Pi 3 is ARM, so the install is slightly different from the AMD64 version:

wget https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-arm64 -O cloudflared
chmod +x cloudflared
sudo mv cloudflared /usr/local/bin/

Then authenticate, create the tunnel, and wire it to the domain:

cloudflared tunnel login
cloudflared tunnel create raspberrypi
cloudflared tunnel route dns raspberrypi str1x3vo.xyz
sudo cloudflared service install
sudo systemctl enable cloudflared
sudo systemctl start cloudflared

At this point - the site is live. Same domain, same URLs, now running off the Pi.

Step 8 - Harden the server

A server sitting on a residential connection needs protecting. Reinstalled everything from the old setup:

Fail2ban - three jails running. SSH bans after 3 failed attempts for 24 hours. Nginx bot scanning and auth brute forcing both covered:

sudo apt install fail2ban -y
sudo systemctl enable fail2ban
sudo systemctl start fail2ban
sudo fail2ban-client status
|- Number of jail: 3
`- Jail list: nginx-botsearch, nginx-http-auth, sshd

SSH key authentication - the same ed25519 key pair from Windows, public key dropped into ~/.ssh/authorized_keys on the Pi. Password login disabled once key auth was confirmed working:

# On Windows - copy public key to Pi
type $env:USERPROFILE\.ssh\id_ed25519.pub | ssh [USERNAME]@[PI_IP] "cat >> ~/.ssh/authorized_keys"

# On Pi - disable password login
sudo nano /etc/ssh/sshd_config
# Set: PasswordAuthentication no
sudo systemctl restart ssh

The result

The gaming PC is no longer hosting anything. The site runs 24/7 off a credit card sized board using 3 watts, in the corner of my room, completely silent. Whether my PC is on, off, or mid-game - the site stays up.

Same domain. Same URLs. Same everything from the outside. Just a much better setup underneath.
