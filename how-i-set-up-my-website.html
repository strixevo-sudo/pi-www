How I Set Up My Self-Hosted Website From Scratch

22 MAR 2026
5 MIN READ
BY STR1X3VO

I wanted my own website. Not a Wix site, not a Squarespace subscription, not paying GBP 10 a month to host a few HTML files. I wanted something I actually owned and controlled, running off my own hardware.

So I built one. From scratch. Off my bedroom PC. And it cost me GBP 1.97.

GBP 1.97 Total cost - domain name only

Here's exactly how I did it.

Step 1 - Setting up the OS

I already had Windows on my main PC so I installed Kali Linux inside VirtualBox - a free virtualisation tool that lets you run a second operating system inside a window. Kali isn't really designed as a server OS but it works fine for this.

The important thing in VirtualBox is to set the network adapter to Bridged mode - this gives the VM its own IP address on your real network instead of hiding behind NAT. Without this, nothing outside the VM can reach your server.

Step 2 - Installing Nginx

Nginx is the web server - it's the software that actually serves your HTML files to people who visit your site. Installing it on Kali is one command:

sudo apt install nginx -y
sudo service nginx start

Once it's running, visiting http://your-local-ip in a browser shows the Nginx welcome page. That means it's working.

Step 3 - The CGNAT problem

Here's where most people get stuck. My ISP is YouFibre, and like a lot of modern ISPs they use something called CGNAT - Carrier Grade NAT. Basically multiple customers share the same public IP address, which means you can't port forward. You literally cannot open a port on your router to let people connect to your home server.

Normal solution: pay for a static IP. I didn't want to do that.

// What is CGNAT Instead of giving every customer their own public IP address, your ISP puts multiple customers behind one shared IP. This saves them money but breaks port forwarding completely.

Step 4 - Cloudflare Tunnel

The solution is Cloudflare Tunnel. Instead of people connecting directly to your PC, your PC connects outbound to Cloudflare's network, and Cloudflare handles all the incoming traffic. No open ports needed at all.

Installing it:

wget https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64 -O cloudflared
chmod +x cloudflared
sudo mv cloudflared /usr/local/bin/

Then log in to your Cloudflare account and create a tunnel:

cloudflared tunnel login
cloudflared tunnel create mytunnel
cloudflared tunnel run mytunnel

To make it start automatically on boot:

sudo cloudflared service install
sudo service cloudflared start

Step 5 - Getting a domain

I bought str1x3vo.xyz from Porkbun for GBP 1.97. That's the only money I spent on this entire project.

Once you have a domain, go to Cloudflare, add it as a site, and update the nameservers at Porkbun to point to Cloudflare. This can take a few hours to propagate.

Step 6 - Connecting everything

Once Cloudflare has your domain active, connect the tunnel to it:

cloudflared tunnel route dns mytunnel str1x3vo.xyz

That's it. Cloudflare handles the SSL certificate automatically. Your site is now live at https://str1x3vo.xyz - served from your bedroom PC, with HTTPS, for free.

The result

A fully self-hosted website, running 24/7 off my home PC, on my own domain, with HTTPS. Zero monthly fees. The only cost was the domain name - GBP 1.97 for a year.

If I can do it with no prior experience, so can you.
