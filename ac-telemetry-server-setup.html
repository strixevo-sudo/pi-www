How to Log AC Telemetry and Send It to a Friend for Analysis

27 MAR 2026
6 MIN READ
BY STR1X3VO

A mate of mine - we'll call him Clyde - wanted help getting his Assetto Corsa setup dialled in. The problem: he's on the other side of the internet. No port forwarding, no shared network. The solution ended up being simple - he ran a telemetry logger during his session, saved everything to a CSV, and sent it over. I analysed it, tweaked his setup, and sent it back. This is how we did it.

What You Need

The person logging their session needs two things:

  • Assetto Corsa installed and running
  • Python 3 installed

That's it. The logger reads AC's built-in shared memory and writes everything to a CSV file. No mods, no apps, no plugins.

Step 1 - Install Python

If they don't have Python, the easiest way on Windows is winget. Open CMD and run:

winget install Python.Python.3.12

Close and reopen CMD after it finishes so the PATH updates. Confirm it worked:

python --version
// Note If winget isn't available, install from python.org. Make sure to tick "Add Python to PATH" during install - otherwise nothing will work.

Step 2 - Get the Logger

Send them the ac_logger.py file. They can drop it anywhere - Desktop works fine. No dependencies to install, just Python.

Step 3 - Run It

Before launching AC, open CMD, navigate to wherever the file is, and run:

python ac_logger.py

The logger sits in the background. It detects when AC goes live automatically and starts writing data. When the session ends, it saves the CSV.

// Important Start the logger before launching AC, not after. It watches for AC to start and begins logging from the moment the session is live.

Step 4 - Find the CSV

Session files are saved in the same folder as the script, named by date, car and track:

2026-03-26_23-54-44_swarm_lewi_altezza_kb3k__saidai_koritsu.csv

One file per session. Send it over - Discord, WeTransfer, whatever works.

What's In the CSV

The logger captures everything useful for setup analysis at high frequency:

  • Speed, gear, RPM, throttle, brake, steering angle
  • Sideslip angle, yaw rate and oversteer index
  • Tyre temps - inner, mid and outer per corner
  • Tyre pressures, loads and wear per corner
  • Suspension travel and dynamic camber per corner
  • Ride height, CG height, turbo boost, air and road temp

That's enough to diagnose tyre temps, suspension behaviour, angle consistency - everything you need to give real setup feedback instead of guessing.

What Went Wrong With Clyde

Here's what tripped him up so you can skip it:

  • Python wasn't installed - winget sorted it in one command
  • He launched AC before starting the logger - missed the early session data. Start the logger first.
  • Wasn't sure where the CSV was saved - it's in the same folder as the script

Once the workflow clicked, it was easy. Session done, CSV sent, setup tweaked, send it back. No live connection needed.
